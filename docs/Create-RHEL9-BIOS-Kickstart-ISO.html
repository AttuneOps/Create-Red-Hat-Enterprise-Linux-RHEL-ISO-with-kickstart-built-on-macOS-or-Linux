How to Create RHEL9 BIOS Kickstart ISO

This Attune Automated blueprint for creating an Red Hat Enterprise Linux (RHEL) BIOS Kickstart ISO is a robust tool designed to simplify the process of generating custom bootable ISO images for RHEL installations, specifically tailored for BIOS boot environments. This blueprint automates the creation of ISOs optimized for compatibility with traditional BIOS-based systems.

With this solution, users can effortlessly customize their RHEL installations to meet specific requirements, including pre-configured settings, software packages, and system configurations. It offers an efficient and streamlined approach to deploying RHEL on a wide array of hardware platforms, making it an excellent choice for system administrators and IT professionals seeking automation and standardisation in their deployment processes.

Attune's automated blueprint for RHEL BIOS Kickstart ISOs empowers users to save valuable time and resources, ensuring consistency and reliability in their Linux deployments. This enhancement in productivity and system management capabilities is essential for those managing environments primarily utilising BIOS-based boot configurations.

The following steps will guide you through the manual process.

Step 1 - Clean Build Files

Deletes the build files and kickstart ISO.

Connect via ssh:

ssh {automationworkerlinuxuser}@{automationworkerlinuxnode}

Execute the following script: 

                
DIR="{automationWorkerBaseDirectory}/build-{newOsLinuxNode.fqn}"
if [ -d "${DIR}" ]
then
    rm -rf "${DIR}" && echo "Removed ${DIR}"
else
    echo "No directory to remove."
fi

ISO="{automationWorkerBaseDirectory}/kickstart_{newOsLinuxNode.fqn}.iso"
if [ -f "${ISO}" ]
then
    rm -fv "${ISO}"
else
    echo "No ISO to remove."
fi

Step 2 - Deploy RHEL9 Boot ISO

Deploys a RHEL9 boot ISO.

This file can be downloaded from the following link. You need a Redhat login. https://developers.redhat.com/products/rhel/download#assembly-field-downloads-page-content-61451

Copy the File(s) RHEL_Boot_ISO.unknown to the target node.

Deploy the File(s) here:

{automationWorkerBaseDirectory}/build-{newOsLinuxNode.fqn}, relative to the home directory

Step 3 - Extract RHEL ISO

Extracts the boot ISO.

Execute the following script: 

                
cd {automationWorkerBaseDirectory}/build-{newOsLinuxNode.fqn}

if ls | grep -q rhel*iso
then
    
    ISO="$(echo rhel*iso)"
    echo "ISO is $ISO"
    
    7z x "${ISO}"
    
    rm -v "${ISO}"
else
    echo "ISO is already extracted."
fi

Step 4 - Deploy RHEL9 BIOS Kickstart Config

Deploys the files required for an automated RHEL9 BIOS kickstart.

RHEL9 BIOS boot files for kickstarting RHEL8.

Copy the File(s) RHEL_BIOS_Kickstart_Config.tar to the target node.

{automationWorkerBaseDirectory}/build-{newOsLinuxNode.fqn}, relative to the home directory

Step 5 - Copy RPM CA Certificates from Drop In Directory

Copies any RPM CA Certificates from the drop in directory.

Execute the following script: 

                
DIR="{newOsRpmCaCertificateDirectory}"
if [ ! -d "$DIR" ]
then
    echo "$DIR does not exist, skipping."
else
    cd "$DIR"
    
    NUMBER_FILES=$(ls -1q | wc -l)
    echo "NUMBER_FILES=${NUMBER_FILES}"
    
    if [ "${NUMBER_FILES}" -eq "0" ]
    then
        echo "No files in $DIR, skipping."
    else
        echo "$DIR exists, copying it's contents to {automationWorkerBaseDirectory}/build-{newOsLinuxNode.fqn}/cacerts."
        cp --verbose --recursive ./* "{automationWorkerBaseDirectory}/build-{newOsLinuxNode.fqn}/cacerts"
    fi
fi

Step 6 - Create BIOS Boot ISO

Creates a BIOS bootable ISO containing the kickstart files.

Execute the following script: 

                
cd {automationWorkerBaseDirectory}/build-{newOsLinuxNode.fqn}

mkisofs -o {automationWorkerBaseDirectory}/kickstart_{newOsLinuxNode.fqn}.iso \
  -b isolinux/isolinux.bin -c isolinux/boot.cat \
  -no-emul-boot -boot-load-size 4 -boot-info-table -J -R -V "KS_RHEL" .

Step 7 - Delete Build Directory

Removes the {ksAttuneBaseDir}/build-{targetServer.fqn} directory.

Execute the following script: 

                
DIR="{automationWorkerBaseDirectory}/build-{newOsLinuxNode.fqn}"
if [ -d "${DIR}" ]
then
    rm -rf "${DIR}" && echo "Removed ${DIR}"
else
    echo "No directory to remove."
fi
